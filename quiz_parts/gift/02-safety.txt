::Safe Mode — призначення::
Навіщо вмикати `/safe-mode` перед ризиковими змінами? {
=зміни відкочуються, якщо сесія CLI обривається
~роутер автоматично робить backup на USB
~блокує всі команди крім `print`
#### Safe Mode захищає від самоблокування під час віддаленої роботи: при обриві з'єднання незастосовані зміни скасовуються.
}

::Команда Safe Mode::
Як увімкнути Safe Mode у CLI? {
=/safe-mode
~/system safe-mode enable
~/tool safe-mode on
#### Safe Mode — пряма службова команда root-рівня: `/safe-mode`. Особливо корисна перед мережевими, firewall і VPN змінами.
}

::Backup перед змінами::
Яка команда зберігає backup конфігурації з ім'ям `before-change`? {
=/system backup save name=before-change
~/export backup name=before-change
~/file backup save before-change
#### `/system backup save` створює бінарний backup. Разом із `/export file=before-change` дає два рівні відновлення перед ризиковими діями.
}

::Export конфігурації::
Що робить `/export file=before-change`? {
=експортує конфігурацію у текстовий .rsc файл на роутері
~негайно перезавантажує роутер
~імпортує конфігурацію з файлу
#### `/export` — безпечна команда для документації та аудиту. Текстовий export зручний для перегляду, порівняння та часткового відновлення.
}

::Ризик import::
Чому `/import` вважають дуже небезпечною командою? {
=може масово застосувати зміни з файлу і відрізати доступ
~лише читає файл без змін конфігурації
~працює тільки з логами, не з конфігурацією
#### `/import` виконує команди зі script/export файлу. Одна помилка в файлі може зламати мережу або VPN — завжди робіть backup і safe-mode.
}

::Безпечні команди читання::
Які команди зазвичай безпечні для перегляду без зміни конфігурації? {
=`print`, `monitor`, `find`, `export`
~`add`, `set`, `remove`, `import`
~`reset`, `disable`, `enable`, `import`
#### Команди читання не змінюють стан. Команди `add`, `set`, `remove`, `reset`, `import` — змінюють конфігурацію і потребують обережності.
}

::Небезпечні команди::
Перед якими діями особливо важливі backup і safe-mode? {
=`remove`, `reset`, `set`, `disable`, `enable`, `import`
~`print`, `monitor`, `find`
~`comment`, `export`, `reset-counters`
#### `remove` і `reset` можуть миттєво зламати доступ. `disable` на WAN/VPN/Wi-Fi також ризиковий. Спочатку `print detail`, потім backup/export.
}

::Рекомендований порядок роботи::
Який порядок дій рекомендований перед зміною конфігурації? {
=спочатку `print detail`, потім `export`, потім `safe-mode`, потім одна зміна
~одразу `import`, потім `reset`, потім перевірка
~20 команд пакетом, потім `print`
#### Не виконуйте пакет змін віддалено. RouterOS виконає все — навіть якщо це відріже вам доступ. Одна зміна → перевірка → наступна.
}

::Undo і Redo::
Що робить `/undo` у RouterOS CLI? {
=скасовує останню зміну, якщо це можливо
~видаляє всі firewall правила
~вимикає Safe Mode
#### `/undo` і `/redo` допомагають відкотити/повторити останню операцію, але не замінюють повноцінний backup перед серйозними змінами.
}

::Команди, яких краще уникати без плану::
Яку команду з переліку краще не запускати без чіткого плану і backup? {
=/system reset-configuration
~/system resource print
~/log print
#### `/system reset-configuration` скидає конфігурацію. Так само небезпечні без плану: `/import`, `/ip firewall filter remove`, `/user remove`, `/certificate remove`.
}
