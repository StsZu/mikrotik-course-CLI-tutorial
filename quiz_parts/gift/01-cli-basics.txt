::Що означає символ / у MikroTik CLI?::
У RouterOS символ `/` на початку команди означає: {
=повний шлях до меню конфігурації CLI
~окрему slash-команду агента, як у Claude Code
~команду негайного перезавантаження роутера
#### У MikroTik `/ip firewall filter` — це шлях до розділу, а не окрема «магічна» команда. Дії `print`, `add`, `set`, `remove` виконуються всередині цього меню.
}

::Приклад повного шляху CLI::
Який з варіантів є коректним повним шляхом до перегляду правил firewall filter? {
=/ip firewall filter print
~firewall filter print
~/ip/firewall-filter/show
#### Повний шлях завжди починається з `/` і повторює ієрархію меню RouterOS: `/ip firewall filter`, потім дія `print`.
}

::Дії всередині меню::
Що робить команда `print` у будь-якому розділі RouterOS CLI? {
=показує список об'єктів у поточному меню
~додає новий об'єкт у конфігурацію
~видаляє об'єкт за номером
#### `print` — базова дія для читання. Вона безпечна і не змінює конфігурацію. Для деталей використовуйте `print detail`.
}

::print проти add::
Яка пара «дія → наслідок» описана правильно? {
=`print` — перегляд; `add` — додавання нового об'єкта
~`print` — видалення; `add` — перегляд
~`print` — зміна параметрів; `add` — експорт
#### `add`, `set`, `remove`, `reset` змінюють конфігурацію. `print`, `monitor`, `find`, `export` зазвичай безпечні для читання.
}

::Пошук команд верхнього рівня::
Як побачити доступні root-команди після введення `/`? {
=натиснути Tab або F1
~ввести `help /` і натиснути Enter
~автоматично з'явиться список без додаткових дій
#### На MikroTik після `/` або `/interface/` натискання Tab або F1 показує доступні підкоманди поточного рівня меню.
}

::Пошук підкоманд у розділі::
Що треба зробити, щоб побачити підкоманди в `/ip/ipsec/`? {
=ввести `/ip/ipsec/` і натиснути Tab
~виконати `/ip/ipsec help`
~виконати `/ip/ipsec list`
#### Той самий принцип, що й на root-рівні: перейдіть у меню та натисніть Tab або F1 для автодоповнення/підказок.
}

::Основні root-розділи::
Який root-розділ відповідає за IPv4: адреси, DHCP, DNS, firewall, NAT, routes? {
=/ip
~/interface
~/system
#### `/ip` — головний розділ для IPv4-мережі та безпеки. `/interface` — фізичні й логічні інтерфейси, `/system` — адміністрування роутера.
}

::Шлях до Wi-Fi клієнтів::
Яка команда показує таблицю підключених Wi-Fi клієнтів? {
=/interface wifi registration-table print
~/wifi clients print
~/interface wireless registration print
#### У RouterOS 7 Wi-Fi керується через `/interface wifi`. Для перегляду клієнтів — `registration-table print`.
}

::print detail::
Чим `print detail` відрізняється від звичайного `print`? {
=показує розширену детальну інформацію про об'єкти
~видаляє приховані об'єкти з виводу
~експортує конфігурацію у файл на диск
#### `print detail` — рекомендований перший крок перед змінами: більше полів, зручніше для діагностики. Не змінює конфігурацію.
}
